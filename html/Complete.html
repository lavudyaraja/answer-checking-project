AI Evaluation Engine

A sophisticated three-pass approach that isolates each concern at temperature=0 for deterministic, fair output.

1

Vision Transcription

4,096 max tokens

AI vision model reads handwriting directly into plain text with confidence scores. No OCR layer.

Purpose: Extract handwritten text from images
Output: Raw transcribed text + confidence
2

Semantic Analysis

4,096 max tokens

Text model evaluates conceptual coverage, grammar, keywords, completeness, and factual accuracy.

Purpose: Analyze understanding & concepts
Output: Semantic scores across 5 dimensions
3

Mark Award

4,096 max tokens

Text model translates semantic scores into final marks using rubric and education-level weights.

Purpose: Assign final marks fairly
Output: Final marks + detailed reasoning

SemanticScore Interface (TypeScript)

interface SemanticScore {
  conceptualCoverage: number  // % of core ideas present
  grammarCoherence:   number  // sentence structure + logical flow
  keywordRelevance:   number  // domain terms used correctly
  completeness:       number  // all required parts addressed
  accuracyScore:      number  // factual / logical correctness
  composite:          number  // weighted total — drives mark calculation
}
Level Conceptual Grammar Keywords Complete Accuracy

Alternative — Single-Pass Evaluation

For faster processing, the system also supports single-pass evaluation via src/lib/single-pass/claude-single.ts — direct vision + evaluation in one API call, suitable for quick evaluations where deep analysis isn't critical.

creates Exams
links to Evaluations

contains Questions

model User

model User {
  id        String   @id @default(cuid())
  email     String   @unique
  name      String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  exams       Exam[]
  evaluations Evaluation[]
}

model Exam

model Exam {
  id          String   @id @default(cuid())
  title       String
  description String?
  subject     String?
  createdById String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  createdBy   User?      @relation(fields: [createdById], references: [id])
  questions   Question[]
  documents   Document[]
  evaluations Evaluation[]
}

model Question

model Question {
  id             String   @id @default(cuid())
  examId         String
  questionNumber Int
  questionText   String
  maxMarks       Float
  modelAnswer    String?
  rubric         String?
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  exam        Exam                 @relation(fields: [examId], references: [id], onDelete: Cascade)
  evaluations QuestionEvaluation[]
}

model Document

model Document {
  id          String    @id @default(cuid())
  examId      String
  fileName    String
  fileUrl     String
  fileType    String    // 'pdf' or 'image'
  pageCount   Int
  uploadedAt  DateTime  @default(now())
  processedAt DateTime?

  exam        Exam         @relation(fields: [examId], references: [id], onDelete: Cascade)
  evaluations Evaluation[]
}

model Evaluation

model Evaluation {
  id              String   @id @default(cuid())
  documentId      String?
  examId          String?
  studentName     String?
  totalMarks      Float
  maxMarks        Float
  percentage      Float
  grade           String?
  overallFeedback String
  confidence      Float
  status          String   // 'processing' | 'completed' | 'failed'
  evaluatedAt     DateTime @default(now())
  evaluatedById   String?

  exam                Exam?               @relation(fields: [examId], references: [id])
  document            Document?           @relation(fields: [documentId], references: [id], onDelete: Cascade)
  questionEvaluations QuestionEvaluation[]
  evaluatedBy         User?               @relation(fields: [evaluatedById], references: [id])
}

model QuestionEvaluation

model QuestionEvaluation {
  id                 String  @id @default(cuid())
  evaluationId       String
  questionId         String
  obtainedMarks      Float
  maxMarks           Float
  confidence         Float
  strength           String?
  mistakes           String?
  missingConcepts    String?
  suggestions        String?
  reasoning          String
  pageNumbers        String  // comma-separated
  studentAnswerImage String? // Base64 image data
  studentAnswer      String? // Extracted text

  evaluation Evaluation @relation(fields: [evaluationId], references: [id], onDelete: Cascade)
  question   Question   @relation(fields: [questionId], references: [id], onDelete: Cascade)

  @@unique([evaluationId, questionId])
}

Setup & Deployment

Prerequisites

  • →Node.js ≥ 20.0.0
  • →npm ≥ 10.0.0
  • →PostgreSQL database
  • →Groq API key and/or Anthropic API key

Environment Variables

# Database
DATABASE_URL="postgresql://user:password@localhost:5432/ai_evaluation"

# AI Services
GROQ_API_KEY="your_groq_api_key"
ANTHROPIC_API_KEY="your_anthropic_api_key"

# Optional: Model Overrides
ANTHROPIC_VISION_MODEL="claude-sonnet-4-6"
ANTHROPIC_TEXT_MODEL="claude-sonnet-4-6"

Installation Steps

# 1. Clone repository
git clone <repository-url>
cd ai-evaluation-system

# 2. Install dependencies
npm install

# 3. Database setup
npx prisma db push
npx prisma generate

# 4. Environment configuration
cp .env.example .env.local
# Edit .env.local with your credentials

# 5. Start development server
npm run dev

Production Deployment

# Build
npm run build

# Start
npm start

# Database migration
npx prisma migrate deploy

Conclusion

This AI Evaluation System represents a comprehensive solution for automated assessment of handwritten answer scripts. By leveraging advanced vision language models and a sophisticated three-pass evaluation engine, the system provides accurate, fair, and detailed evaluations across multiple subjects and education levels.

The modular architecture allows for easy customization and extension, while the robust technology stack ensures reliability and performance. With features like real-time progress tracking, comprehensive feedback, and dual AI provider support, this system is well-suited for educational institutions seeking to modernize their evaluation processes.
