How Three-Pass Works

Three sequential API calls — each with a specialized, focused prompt — building on the previous pass.

1

Pass 1 — Vision Transcription 10–15 s

Vision model reads handwriting directly into plain text.

const pass1 = await anthropic.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 2000, temperature: 0,
  messages: [{ role: 'user', content: [
    { type: 'text',  text: 'Transcribe this handwritten answer exactly...' },
    { type: 'image', source: { type: 'base64', ... } }
  ]}]
})
// Output: "The student's handwritten answer..."
2

Pass 2 — Semantic Analysis 8–12 s

Text model evaluates the transcript against the model answer across 5 dimensions.

const pass2 = await anthropic.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 1500, temperature: 0,
  messages: [{ role: 'user', content:
    `Analyze this answer: "${transcription}" against question: "${question}"`
  }]
})
// Output: Semantic scores and analysis
3

Pass 3 — Mark Award 8–12 s

Text model translates semantic scores into numeric marks and detailed feedback.

const pass3 = await anthropic.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 1500, temperature: 0,
  messages: [{ role: 'user', content:
    `Award marks based on semantic analysis...`
  }]
})
// Output: Final marks and detailed feedback

Key Characteristics

  • Three separate API calls per question
  • Specialized prompts for each task
  • Step-by-step processing with error isolation
  • Granular retry capability per pass
  • Temperature set to 0 for consistency

Overall Recommendation — Hybrid Approach

// Recommended implementation strategy
async function smartEvaluation(question, image) {
  try {
    const singleResult = await evaluateSinglePass(question, image)

    // Quality gate
    if (singleResult.confidence > 85 && singleResult.semanticScore.composite > 80) {
      return singleResult
    }

    // Fallback to three-pass for low confidence
    console.log('Low confidence, falling back to three-pass')
    return await evaluateThreePass(question, image)

  } catch (error) {
    console.log('Single-pass failed, using three-pass fallback')
    return await evaluateThreePass(question, image)
  }
}

Benefits of Hybrid Approach

  • Best of both worlds — cost efficiency + quality assurance
  • Adaptive selection — use optimal approach per case
  • Graceful degradation — always have a fallback
  • Continuous improvement — collect data for optimization

Conclusion

Key Benefits (Single-Pass)

  • 3× faster processing speed
  • 25% cost savings per evaluation
  • Simpler architecture, fewer failure points

Trade-offs

  • Minor accuracy reduction (3–4% lower)

  • Less granular error handling
  • More challenging debugging

🏆 Final Recommendation — Hybrid Approach

  1. Primary: Use single-pass for standard evaluations
  2. Fallback: Use three-pass for complex / low-confidence cases
  3. Adaptive: Select approach based on content analysis
  4. Monitoring: Track quality and performance metrics continuously
